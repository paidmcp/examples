# Contributing

Thanks for contributing to `paidmcp/examples`.

## Contribution flow

1. Open an issue for larger proposals.
2. Keep each PR scoped to one example or one concern.
3. Update related docs when commands or behavior change.
4. Run checks in the example you changed:

```bash
npm run check
npm run format:check
```

## PR checklist

- Change summary includes motivation
- Checks pass locally
- No secrets included
- Pricing/payment behavior changes are documented

This is maintained by a solo developer, so reviews may take time.
